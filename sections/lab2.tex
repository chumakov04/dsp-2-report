\section{Lab exercise 2: Notch and Comb Filter Design}

Notch filters are used to filter out specific frequency interference at $\omega_0$. Either one specific frequency imposed by some measurement device or narrow range of frequencies. Thus, notch filters are bandstop. Amplitude response has very narrow notch at frequency range to be blocked with other frequencies pass through unchanged. 

They are created with second order IIR filter designed directly from pole zero plot so that conjugate complex zero is located at that specific frequency $\omega_{0}$. Since zeros are exactly on the unit circle, filter gain at $\omega_{0}$ is zero.

For all other frequencies along unit circle distance from zeros and poles is roughly equal so gain is roughly 1. Conjugate complex poles are placed at the same angle but inside the unit circle at distance $r$ from origin. Pole radius $r$ controls steepness and width of the notch. Notch becomes more steep and narrow as r approaches 1. For stability, $r < 1$ must hold (Figure~\ref{fig:201}).

\begin{figure}[H]
  \includegraphics[width=0.6\textwidth]{image_78.png}
  \caption{notch filter pole zero design basic idea}\label{fig:201}
\end{figure}

Theoretically, this type of filter is even more suitable for lab 2 noise spike attenuation, as it is designed specifically for such narrow frequency range interference removal (between 999 and 1001 Hz).

Comb filters are used to attenuate multiple equally spaced frequencies (harmonics) that are multiple of fundamental frequency of interference. For this purpose, notch filter can be used as comb filter to have multiple repeated notches at harmonics. For example, at 1001, 2002, 3003 Hz.

They are constructed by inserting zeros between original filter coefficients (since notch is IIR second order, if use as comb zeros must be inserted between both $b_n$ and $a_n$ coefficients) which increases sampling frequency $\omega_s$ while keeping the original filter shape unchanged. Basically, this means that instead of one notch filter operating up to $\omega_s/2$, there will be multiple, depending on the amount of zeros inserted. 

Each inserted zero extends frequency range by original $\omega_s/2$, so inserting $n$ zeros produces ($n+1$) copies of filter shape between 0 to $\omega_s/2$ and replaces $z^{-1}$ with $z^{-(n+1)}$ in the transfer function (Figure~\ref{fig:202}).

\subsection{Laboratory 2 -- Design a Notch filter using the Pol-Zero diagram for a stop frequency fs=1000Hz and a sample rate fa=10000Hz. Use as the pole radius r=0.98.}

Following theoretical introduction above, in transfer function (\ref{eq:notch-tf}) numerator zeros $e^{\pm j\omega_0}$ are on the unit circle and create zero gain at $\omega_0$. Denominator poles $r * e^{\pm j\omega_0}$ are inside the unit circle at radius $r$. 

It is further simplified to form with cosine of angular frequency $\omega_0$ that I used in function for notch filter.

\subsubsection{(a) Write a function to design a Notch filter with the following input parameters:}

% given on the exercise sheet:
\begin{itemize}
  \item stop frequency fs; pole radius r; sampling rate fa
\end{itemize}
With
\begin{equation}
  H(z) = \frac{(z - e^{j\omega_0})(z - e^{-j\omega_0})}
              {(z - re^{j\omega_0})(z - re^{-j\omega_0})}
       = \frac{z^2 - 2z\cos(\omega_0) + 1}
              {z^2 - 2zr\cos(\omega_0) + r^2}
  \label{eq:notch-tf}\tag{1}
\end{equation}
and
\begin{equation}
  \omega_0 = 2\pi\frac{f_s}{f_a}
  \label{eq:w}\tag{2}
\end{equation}

\begin{figure}[H]
  \includegraphics[width=0.48\textwidth]{image_27.jpg}\hspace{4pt}
  \includegraphics[width=0.48\textwidth]{image_28.jpg}
  \caption{notch filter frequency response and pole-zero map}\label{fig:203}
\end{figure}

I wrote \hyperlink{code:notchFilter}{notchFilter} function that computes normalized angular frequency $\omega_0 = 2\pi \cdot f_s / f_a$ given desired parameters of $f_s$, $f_a$, $r$ and returns $b_n$ and $a_n$ coefficients. 

For $f_s = 1000$ Hz, $f_a = 10000$ Hz, $r = 0.98$ output is (Figure~\ref{fig:204}).

\begin{figure}[H]
  \includegraphics[width=0.6\textwidth]{image_79.png}
  \caption{notch filter coefficients}\label{fig:204}
\end{figure}

Figure~\ref{fig:203} shows very steep notch magnitude at 1 kHz reaching -30.18 $dB$ damping effect, which attenuates desired frequency. In pole zero map, as expected, zero is located at $\omega_0$ frequency to attenuate it with pole located almost where zero is with $r = 0.98$, this is what influences steepness of filter transition. Stability constraint is not violated $r$ < 1, thus system is stable and filter must work correctly. Phase changes at $\omega_0 = 1 kHz$ between -89 and +89 degrees because zeros on unit circle cause sign change in transfer function G(z), equivalently vector pointing to $\omega_0$ frequency passes through zero point changing its sign by 180 degrees (Figure~\ref{fig:205}).

\begin{figure}[H]
  \includegraphics[width=0.6\textwidth]{image_83.png}
  \caption{phase sign change at zero crossing}\label{fig:205}
\end{figure}

To produce magnitude and phase responses I used freqz(b,a,n,fs) Matlab function plots the frequency response of the filter at $n$ sample points for sample rate $f_s$ \footnote{\url{https://www.mathworks.com/help/signal/ref/freqz.html\#bt8l4c4\_sep\_shared-ba}}.

\codeused
\hypertarget{code:notchFilter}{}
\begin{matlab}
fs = 1000; fa = 10000; r = 0.98;
function [b, a] = notchFilter(fs, fa, r)
    % normalized angular frequency
    w0 = 2 * pi * fs / fa;
    % b numerator coefficient applied to input
    b = [1, -2 * cos(w0), 1];
    % a denominator coefficient comes from feedback
    a = [1, -2 * r * cos(w0), r^2];
end
[b, a] = notchFilter(fs, fa, r);
resolution = 1024; freqz(b, a, resolution, fa); zplane(b, a);
\end{matlab}

NB: for sections (b) and (c) frequencies are selected arbitrary with no bias towards some value. Roughly multiples of each other to display behavior as values change significantly enough at each step.

\subsubsection{(d) Correct the gain factor for your notch filter function.}

Without correction passband gain is only largely constant but not exactly 1. Gain 1 is not guaranteed theoretically. 

Only $b$ coefficients are safe to scale, because scaling $a$ coefficients can put pole outside of unit circle violating stability constraint of $r$ < 1 that makes system unstable. Scaling $b$ coefficients just multiplies G(z) transfer function by constant factor.

Since $z = e^{j\omega}$ at $\omega = 0$ (DC component of 0 H), $z = e^{j * 0} = 1$. So, we need gain correction factor (in my code it is $b_0$) that is ratio of denominator to numerator evaluated at $z = 1$, thus scales $b$ coefficients so that $H(z = 1) = 1$. 

\codeused
\hypertarget{code:notchFilterGC}{}
\begin{matlab}
function [b, a] = notchFilterGC(fs, fa, r)
    w0 = 2 * pi * fs / fa;
    b = [1, -2 * cos(w0), 1];
    a = [1, -2 * r * cos(w0), r^2];
    % make H(z = 1) = 1
    b0 = abs(sum(a)) / abs(sum(b));
    b = b0 * b;
end
\end{matlab}

\subsection{Laboratory 2 -- Design a Comb filter out of a Butterworth low-pass filter with 3 repetitions}

Basic comb filter with $r$ = 0.9999995 and $Lf$ = 9 uses FIR structure with $a = 1$ so only zeros, no poles. From the output in figure 29 it can be seen that 9 zeros are placed inside the unit circle with visually equal space between them. As expected, 9 repeated notches are produced in magnitude spectrum from 0 to $pi$ in normalized frequency.

\begin{figure}[H]
  \includegraphics[width=0.48\textwidth]{image_35.jpg}\hspace{4pt}
  \includegraphics[width=0.48\textwidth]{image_36.jpg}
  \caption{basic comb filter frequency response and pole-zero map}\label{fig:210}
\end{figure}

Butterworth low pass should be used as comb with n = 3 repetitions. As explained in theoretical part, sampling frequency $\omega_s$ remains the same but instead of 1 low pass filter shape from 0 to $\omega_s/2$ there must be 3, compressing original range of low pass with a factor of 3. This principle is exactly same as in Figure~\ref{fig:202}, but there range is compressed with a factor of 5.

\subsubsection{(a) Write a Comb filter function which can design a Comb filter by taking the following parameters:}

I designed \hyperlink{code:combFilter}{function for comb filter} which is very simple. First, it initializes array of zeros for both $b$ and $a$ coefficients, which has a desired length after $n$ zeros insertion for repeating filter shape. After, we just rewrite every n $th$ position with original filter coefficients, next that would be Butterworth lowpass.

% given parameters
\begin{itemize}
  \item filter coefficients b; filter coefficients a; repetitions n
\end{itemize}

\codeused
\hypertarget{code:combFilter}{}
\begin{matlab}
% 9 zeros part
r = 0.9999995; Lf = 9;
b = [1 zeros(1, Lf - 1) -r^Lf]; a = 1;
% custom function
function [b_comb, a_comb] = combFilter(b, a, n)
    % length(b) - 1 = 2 gaps between coefficients
    % [b0, 0, 0, b1, 0, 0, b2] length is +1 then
    b_comb = zeros(1, (length(b) - 1) * n + 1);
    a_comb = zeros(1, (length(a) - 1) * n + 1);
    % original coefficients every n th position
    b_comb(1:n:end) = b;
    a_comb(1:n:end) = a;
end
\end{matlab}

\subsubsection{(b) Use your Comb filter function and analyse it with the amplitude and phase response of the filter.}

I used order $N$ = 4 butterworth lowpass with cutoff frequency at $0.2\pi$. After $0.2\pi$ normalized frequency filter must start attenuation. This was an arbitrary choice during the lab, not biased towards a certain value. Since order is 4, there must be 4 zeros and 4 poles in transfer function, also in pole zero map.

\begin{figure}[H]
  \includegraphics[width=0.32\textwidth]{image_37.jpg}\hspace{4pt}
  \includegraphics[width=0.32\textwidth]{image_38.jpg}\hspace{4pt}
  \includegraphics[width=0.32\textwidth]{image_39.jpg}
  \caption{butterworth lowpass, comb filter with n=3, and pole-zero map}\label{fig:211}
\end{figure}

After applying comb function with $n = 3$, 3 zeros are inserted between each coefficient $b$ and $a$. Theoretically expected for butterworth lowpass filter shape to be repeated 3 times across spectrum 0 to $\pi$ with original lowpass filter shape to be compressed from 0 .. $\pi$ to 0 .. $\pi/3$.

Figure~\ref{fig:211} output confirms theoretically correct behavior. Magnitude spectrum of butterworth lowpass as comb shows that original filter shape was compressed 0 .. $\pi/3$ with attenuation (cutoff frequency) now starting at 0.04/5 $\pi$ roughly. Phase is also compressed to 0 .. $\pi/3$, which is $1/3$ of original lowpass filter range between 0 and $\pi$. Poles and zeros are copied 3 times, so there are now 12 poles and 12 zeros. 

Principle is very straightforward, such that the only effect that happens is 3 repeated copies of lowpass butterworth are placed inside original frequency range from 0 to $\omega_s/2$, preserving original characteristics, just being compressed.

\codeused
\begin{matlab}
% order is 4, cutoff is 0.2 
[b_lp, a_lp] = butter(4, 0.2);
% comb with 3 repetitions
[b_comb, a_comb] = combFilter(b_lp, a_lp, 3);
freqz(b_lp, a_lp, 1024); freqz(b_comb, a_comb, 1024);
zplane(b_comb, a_comb);
\end{matlab}

\subsection{Homework 2 (Implement your own Notch-Comb filter function)}

To design \hyperlink{code:notchCombFilter}{notch comb filter} function given sample rate $f_a$, repetitions $n$, pole radius $r$, I first compute stop frequency $f_s$ of notch to be repeated, which is $f_a/2$, that corresponds to range between 0 and $\omega_{s}/2$, divided by $n$ repetitions of original filter shape. Then apply \hyperlink{code:notchFilterGC}{notch filter} function to create notch filter with given pole radius parameter. And finally feed computed by notch filter $a$ and $b$ coefficients into \hyperlink{code:combFilter}{comb filter} function to get notch as comb filter.

Final notch comb filter coefficients are only 7 for $a$ and 7 for $b$, which is significantly more efficient to FIR, requiring much higher filter order for steep notches (Figure~\ref{fig:213}).

\codeused
\hypertarget{code:notchCombFilter}{}
\begin{matlab}
function [b_comb, a_comb] = notchCombFilter(fa, n, r)
    % notch at fa/2 divided by n repetitions
    fs = fa / (2 * n);
    % gain corrected notch
    [b, a] = notchFilterGC(fs, fa, r);
    % repeat n times with z -> z^n
    [b_comb, a_comb] = combFilter(b, a, n);
end
fa = 10000; n = 3; r = 0.98;
[b_notchcomb, a_notchcomb] = notchCombFilter(fa, n, r);
\end{matlab}
