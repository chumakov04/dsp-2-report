\section{Lab exercise 4: 2D-Filter and Image Processing}

Image $P(x, y)$ is 2 dimensional signal with spatial coordinates $x$ and $y$. Signal values in this case are intensity (brightness) values of pixels ranging from 0 to 255 captured by sensor. Some captured by digital sensor rays are very strong and cause maximum intensity 255, equivalenty to white color. Some areas on the sensor do not capture any rays and thus can get intensity value of 0, equivalently complete black color. 

2 dimensional signals can be modified and filtered in time and frequency domains following same principles as 1 dimensional signals by applying DFT in both spatial directions $x$ and $y$ sequentially, first DFT in $x$ direction (column by column), and second applied to the result of first in $y$ direction (row by row). The result it $Q(u, v)$ frequency spectrum that has same size as image with DC component at $Q(0, 0)$ representing average intensity of image, and values corresponding to bigger $u$ or $v$ represent high frequency details in image like tiny edges. Of course, conjugate symmetry applies but with 4 quadrants in frequency spectrum, as all properties for 1 dimensional DFT hold. Each $(u, v)$ pair is information about containment of sinusoid with that specific frequency in image (Figure~\ref{fig:401}).

\begin{figure}[H]
  \includegraphics[width=0.6\textwidth]{image_103.png}
  \caption{frequency spectrum}\label{fig:401}
\end{figure}

Labs 4 and 5 do not focus on frequency spectrum of images, instead use 2 dimensional kernels, thus I will not go into much more detail. What is important to mention is that frequency in image is basically amplitude change between pixel intensities of neighboring pixels (Figure~\ref{fig:402}). If 1 pixel has value 0 = Black, and second pixel receives ray with value 255 = White, then this is highest frequency possible. Amplitude change is from lowest to highest value (white edge on black backround). While, smooth changes like sky gradient would correspond to low frequency components. 

\begin{figure}[H]
  \includegraphics[width=0.48\textwidth]{image_102.png}
  \caption{high frequency amplitude change}\label{fig:402}
\end{figure}

Same logic as with 2 dimensional DFT applies also to 2 dimensional convolution, which inherits properties of 1 dimensional convolution that FIR filtering with 2 dimensional kernels in $x$ and $y$ direction use. Image (2 dimensional signal) is convolved (compute inner product) with FIR filter 2 dimensional signal (coefficients) sliding across the image. Since 1 dimensional convolution goes only into 1 direction, 2 dimensional as DFT goes in both (Equation~\ref{eq:conv2d}).

\begin{equation}
P_{x,y} * F_{x,y} = \sum_{n=0}^{N} \sum_{m=0}^{M} F_{m,n} \cdot P_{x-m, y-n}
\label{eq:conv2d}
\end{equation}

$P_{x,y}$ is image signal, $F_{x,y}$ is FIR filter window. Same as 1 dimensional convolution theorem that says that convolution in time domain corresponds to multiplication in frequency domain, same applies to image filtering. Labs 4 and 5 do not use frequency spectrum based filtering, thus I will not go into further detail.  Basic idea behind frequency domain filtering is instead of guessing values for perfect window FIR filter mask in spatial domain, simple rule for mask can be specified for frequency domain pairwise multiplication, like distance from origin to $(u, v)$ pair. Lowpass sets frequencies to 0 if distance is large, corresponding to attenuation of high frequency components.

Basic principle of filtering with spatial domain image kernels shown on mean window (Figure~\ref{fig:403}). With 3 by 3 kernel boundary pixels of image require padding of 1 which is done using different methods, depending on what is desired, such as padding with constant vakue replication padding, reflection mirror padding, or zero padding \footnote{\url{https://www.mathworks.com/help/images/imfilter-boundary-padding-options.html}}. Central pixel in 3 by 3 neighborhood gets average value. Convolution window continue sliding until all pixels in image receive average value. 

\begin{figure}[H]
  \includegraphics[width=0.6\textwidth]{image_105.png}
  \caption{mean kernel}\label{fig:403}
\end{figure}

Another kernels that I use in the lab 4 are median, laplacian, gaussian, and combined laplacian of gaussian.

Gaussian kernel is better version of mean window, because it gives more weight towards center pixel and direct neighborhood pixels. For example, in 3 by 3 kernel corner pixels receive weight = 1. This also helps to preserve object shape and edges better. Weight is controlled by $\sigma$ variable (Figure~\ref{fig:409}), such that smaller $\sigma$ results in more weight towards central pixel of convolution mask, and less towards neighborhood.

Median kernel replaces current center pixel with median value across its neighborhood. It is better than mean for noise reduction, because it removes small spikes caused by some reflections like stars on sky without blurring object edges so much as average does.

Laplacian uses kernel values to determine image gradient in pixel intensity values in $x$ and $y$ direction (Figure~\ref{fig:408}). Instead of gradient vector $\nabla f(x,y) = g_x(x,y), \; g_y(x,y)$ that stores change in $x$ and $y$ directions separately, laplacian produces scalar by calculating horizontal and vertical second derivatives and adding together (Equation~\ref{eq:laplacian}). 

\begin{equation}
\nabla^2 f(x,y) = \frac{\partial^2 f(x,y)}{\partial x^2} + \frac{\partial^2 f(x,y)}{\partial y^2}
\label{eq:laplacian}
\end{equation}

If there is a change, edge pixels receive amplified brightness values. Along with tresholding applied it can draw object edges while making everything else completely black. 

\begin{figure}[H]
  \includegraphics[width=0.48\textwidth]{image_111.png}
  \caption{second derivative operator}\label{fig:408}
\end{figure}

Laplacian values are selected to represent second derivative, making it vulnurable to detect noise spike as real object edge. Therefore, gaussian is used together with laplacian to remove such noise from image, also called mexican hat operator (Figure~\ref{fig:409}).

\begin{figure}[H]
  \includegraphics[width=0.48\textwidth]{image_112.png}
  \caption{second derivative operator}\label{fig:409}
\end{figure}

There are simplier methods to manipulate images are based on pixel intensity values distribution. Histogram tells how much pixels from 0 (dark) to 255 (bright) are contained in image, equivalently histogram provides high level summary of image based on frequency of each brightness value in image. If image histogram truns out to be bipolar with 2 dominant peaks (a lot of bright and dark pixels), then simple tresholding could be used to assign 0 = Black for 1 group and 1 = White to another (Figure~\ref{fig:404}).

\begin{figure}[H]
  \includegraphics[width=0.6\textwidth]{image_104.png}
  \caption{image histogram}\label{fig:404}
\end{figure}

Another method of histogram modification is contrast enhancement that can be achieved with transfer functions and histogram equalization. 

Contrast is how well spread intensity values are in image, which is connected to how human eye perceives surroundings. If image is bright everywhere, it is difficiult to see tiny details, as well as in complete dark image. In low contrast image histogram brightness values are clustered in a narrow range like most between 200-210 intensity levels, while other before and after this cluster are not used well. Thus, transfer function can be designed to spread out this narrow range to 0-255.

More formally, transfer function $T$ maps input intensity $r$ to output $s = T(r)$. Slope thus is multiplier of differences between pixel values. Where slope $>1$, input intensity range is stretched to wider output range and contrast increases. Where slope $<1$, range is compressed and contrast decreases (Figure~\ref{fig:405}). 

Gamma correction function uses exactly this principle. It is non linear transfer function to improve contrast in specific intensity range while compressing others. Output value is $s=g(x,y)$, input value is $r=f(x,y)$ which is original intensity value of pixel. Transfer function is $T[f(x,y)] = (f(x,y))^p$ applied to normalized pixel values in $[0, 1]$ range, where exponent $p$ determines shape of curve (Figure~\ref{fig:405}). If, for example, $p = 1$, $T$ is identity function with slope 1 everywhere, no change occurs. Only change in Matlab is naming convention $\gamma$ instead of $p$.

\begin{figure}[H]
  \includegraphics[width=0.6\textwidth]{image_106.png}
  \caption{transfer function and gamma correction}\label{fig:405}
\end{figure}

In probability density function $p_r(r)$ (chance of each brightness value occurance) total area under curve is always 1 because it represents all pixel intensity values. If section is stretched by slope $> 1$, width increases so height must decrease to keep area same. If compressed by slope $<1$, width decreases and height increases (Figure~\ref{fig:406}).

Histogram equalization achieves best possible contrast in image by using image intensity values histogram $p_r(r)$ to calculate custom transfer function that makes output histogram flat in density $p_s(s)$, equivalently uniform distribution with equal prbability of each intensity level occcuring. Cumulative distribution function needs to be applied as intensity transfer function. In continious case with infinitely many brightness leveks this is integration of $p_r(r)$. 

\begin{figure}[H]
  \includegraphics[width=0.46\textwidth]{image_108.png}\hspace{4pt}
  \includegraphics[width=0.50\textwidth]{image_109.png}
  \caption{continious histogram equalization}\label{fig:406}
\end{figure}

For discrete 8-bit image with $L = 256$ gray levels, integration is replaced by discrete summation. Relative frequency of each intensity level $k$ is $p_r(r_k) = \frac{n_k}{n}$, where $n$ is total number of pixels and $n_k$ is number of occurrences of specific gray level $k$. Running sum of these probabilities gives each intensity level $k$ new rank (Equation~\ref{eq:histeq}).

\begin{equation}
s_k = T(r_k) = \sum_{i=0}^{k} p_r(r_i) = \sum_{i=0}^{k} \frac{n_i}{n}
\label{eq:histeq}
\end{equation}

Simple example of discrete histogram equalization for 8-bit image with 0-255 grey levels (Figure~\ref{fig:407}). Exactly such algorithm I used in lab. 

\begin{figure}[H]
  \includegraphics[width=0.6\textwidth]{image_110.png}
  \caption{discrete histogram equalization example}\label{fig:407}
\end{figure}

\subsection{Laboratory 4 -- Perform the following procedure of a automatic histogram equalisation:}

I wrote custom functions to perform histogram equalization based on the described theory and example on Figure~\ref{fig:407}.

\subsubsection{(a) build a histogram of the pixel values}

\begin{figure}[H]
  \includegraphics[width=0.6\textwidth]{image_51.jpg}
  \caption{histogram of pout.gif pixel values}\label{fig:408}
\end{figure}

First of all I store image as 1 dimensional array of brightness values to compute histogram based on their frequency (count). Output on Figure~\ref{fig:408} shows that most pixel values are largely concentrated between 70-150 intensity values range, which indicates poor contrast. As already mentioned in theoretical intoduction, values cluttered need to be spread out through whole 0 to 255 value range with enough space inbetween to produce a better image contrast.

\codeused
\begin{matlab}
i = imread('pout.gif');
image = i(:);
% pixel values from 0 to 255
histogram = zeros(1, 256);
for idx = 1:length(image)
    % h(1) count of pixel value 0
    % for ex, at idx = 3, pixel value 107 appears again -> h(108) goes from
    % 1 to 2 count
    histogram(image(idx) + 1) = histogram(image(idx) + 1) + 1;
end
% plot histogram
figure
bar(0:255, histogram); title('histogram'); xlabel('pixel value'); ylabel('count');
\end{matlab}

\subsubsection{(b) build the cumulative sum of this histogram}

I build cumulative sum vector by summing all histogram counts from 0 to $k$ at each position $k$. First element stays same as histogram, then each next adds previous cumulative value and current histogram count. Last value $cumulative(256)$ trivially equals total pixel count in image 69480.

\codeused
\begin{matlab}
cumulative = zeros(1, 256);
% pixel value 0 same
cumulative(1) = histogram(1);
% for each pixel value level between 1 and 255
for k = 2:256
    % old cumulative value + count of h
    cumulative(k) = cumulative(k - 1) + histogram(k);
end
\end{matlab}

\subsubsection{(c) normalise this cumulative sum to the max. pixel value}

Normalize calculated vector by dividing cumulative sum by its last value $cumulative(256)$ which equals total number of pixels. Result is 256 element vector with values from 0 to 1, which is exactly the cumulative distribution function $s_k = T(r_k)$ from Equation~\ref{eq:histeq}. Output on Figure~\ref{fig:409} shows that normalized cumulative sum is a very steep transfer function (will be used as lookup table) that maps cluttered 70-150 intensities value range to 0-1 taking all that range on $y$ axis.

\codeused
\begin{matlab}
% normalized 1*256 vector -> 256 values
% last value guaranteed to be max value
normalized = cumulative / cumulative(256);
figure; plot(0:255, normalized); title('cumulative sum of histogram');
\end{matlab}

\begin{figure}[H]
  \includegraphics[width=0.6\textwidth]{image_52.jpg}
  \caption{normalized cumulative sum of histogram}\label{fig:409}
\end{figure}

\subsubsection{(d) use this as a mapping from old-new pixel values (lookup-table)}

For each pixel in image, old pixel value is used as index into normalized cumulative sum, and result is multiplied by 255 to get new pixel value. For example, pixel with old value 107 looks up {normalized(108) = 0.31}, multiplied by 255 this gives new value 79. 

Output on Figure~\ref{fig:410} shows resulting resulting output (mapping) vector. These are values of equalized image flattened 1 dimensional vector.

\begin{figure}[H]
  \includegraphics[width=0.6\textwidth]{image_114.png}
  \caption{resulting vector that performs mapping and scaling by 255}\label{fig:410}
\end{figure}

This is exactly what I explained with example of discrete histogram equalization from Figure~\ref{fig:407}, where running sum stretches cluttered intensity region by creating gaps between mapped values.

\codeused
\begin{matlab}
% initialize mapping as copy of image vector
% output is 69480*1 uint8 column vector
output = image;
for k = 1:length(image)
    % old_pixel = 107
    % old_pixel + 1 = 108
    old_pixel = image(k);
    % normalized(108) = 0.31
    % * 255 = 79 -> new pixel value
    output(k) = normalized(old_pixel + 1) * 255;
end
\end{matlab}
